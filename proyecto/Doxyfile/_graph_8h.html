Calcula la mejor ruta y el costo de dicha ruta. @Se hace uso del algoritmo Dijkstra y el algoritmo BFS (Breadth First Search)

===============================================================

======================{ METODOS PUBLICOS }=====================

Parameters
origenReferencia a un objeto Station con el que comenzaremos nustro algoritmo.
destinoReferencia a un objeto Station con el que terminaremos nuestro algoritmo.
stationsReferencia a una DLL donde se almacenan todos nuestros nodos representados por ciudades.
rootsReferencia a una DLL donde se almacenan todas nustras aristas junto con sus pesos.
pathCadena de texto que nos permitira mostrar la mejor ruta y el costo final.
lenTamaño de la cadena de texto que usamos para mostrar la mejor ruta y el costo final.

Se crea el mapa

Se obtiene el nombre del nodo inicial

Se agregan todas los nodos (ciudades) al mapa

Se hace la busqueda del nodo origen en el mapa

Se inserta el nodo origen en una cola

Mientras la cola no esté vacia se genera todo el algoritmo

Se extrae una ciudad de la cola

Se castea a un objeto Station

Se obtienen todos sus vecinos

Se itera sobre todos sus vecinos

Se castea cada vecino a un objeto Station

Se obtinen todas las aristas del grafo

Se itera sobre todas las aristas del grafo

Se castea cada arista a un objeto Roots

Se compara hasta obtener

la arista buscada y así saber su peso

Se valida que el vecino econtrado no haya sido visitado

y que el peso sea el menor entre todas las aristas

De ser así se asume que es la mejor ruta para continuar

y que el vecino es el mejor cadidato entre todos los vecinos

Se busca en la tabla y se castea al mejor vecino

Se colorea al mejor vecino como descubierto

Se suma el peso final

Se reinicia el menor peso

Se concatena el nombre del vecino a la mejor ruta

Todo el ciclo While depende de que haya elementos en la cola

no se agregan elementos a la cola si ya llegamos al nodo (ciudad) destino

Al finalizar el ciclo While se ha generado
